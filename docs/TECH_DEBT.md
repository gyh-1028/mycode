# MyCode 活跃技术债

> 更新日期：2026-07-14。这里只保留仍需处理的事项；已关闭历史由 Git 记录。

## TD-01 模型目录发布前复核

- **优先级：P0 / 发布门禁**
- **现状：** 模型目录已迁移到 `src/mycode/data/model_catalog-v1.json`，具备 schema、目录版本、
  模型验证时间和价格验证时间；当前两个时间均为空。
- **风险：** Provider 的模型 ID、停用日期、思考参数和价格会变化。过期数据会导致请求失败或
  成本门禁失真。
- **处理：** 对照各 Provider 官方文档逐项复核；记录 UTC 日期和来源；未知价格保持缺失，
  不用猜测值填充。
- **验收：** `mycode doctor` 显示有效验证日期；所有默认模型做 API smoke；真实 Eval 预算可计算。

## TD-02 真实模型基线缺失

- **优先级：P1 / P13-P14 验收门禁**
- **现状：** `safe-core-v1` 20 个、`host-repair-v1` 6 个、`codeintel-v1` 12 个 case 和离线测试已存在，
  但没有真实模型三次重复结果。
- **风险：** 无法量化任务成功率，也无法证明自动上下文带来至少 10 个百分点提升。
- **处理：** 固定模型、温度、预算和目录版本，分别运行基线与候选；运行会产生 API 费用，
  不纳入普通 CI。
- **验收：** 保存 run ID、`pass@1`、`pass@3`、成本、延迟和 token；完成同模型 A/B 报告。

## TD-03 大仓库冷索引性能

- **优先级：P1**
- **现状：** 100 文件增量更新约 `0.72s`，已低于 `2s` 目标；本机非 Git 10,000 小文件冷索引
  约 `72-98s`，未达到 `30s` 目标。
- **已做：** 单事务批处理、连接及时关闭、父目录解析缓存、Git change-set 快速路径；超过
  2,000 个 tracked 文件时可使用 `git cat-file --batch` 读取 clean blob。
- **处理：** 在真实 10,000 文件 Git 仓库复测；按阶段记录发现、读取、AST 和 SQLite 耗时；
  再决定是否使用持久后台索引进程。
- **验收：** 冷索引 `<30s`，100 个变更文件 `<2s`，结果由 `scripts/benchmark_codeintel.py` 输出 JSON。

## TD-04 大型模块拆分

- **优先级：P1**
- **热点：** `cli.py`、`agent/runner.py`、`tui/app.py`、`server/session.py`、Web `App.tsx`。
- **风险：** 命令分发、运行逻辑和 UI 状态集中，增加修改冲突和局部测试成本。
- **处理：** 先按稳定协议拆命令 handler、RPC domain handler 和 React hooks；不改变外部行为。
- **验收：** 现有契约测试不变；每个拆分模块具有独立单测；不新增全局状态。

## TD-07 视觉回归仍是结构断言

- **优先级：P2**
- **现状：** Playwright 会检查布局区域、溢出、重叠、图标可访问名称并输出截图，尚未做像素基线比较。
- **风险：** 细微颜色、间距或字体回归可能不触发结构断言。
- **处理：** 在 Linux CI 固定字体和浏览器后再引入稳定截图基线，避免跨平台误报。

## TD-08 安全层不是沙箱

- **优先级：持续项**
- **现状：** 路径、symlink、敏感文件、命令黑名单和审批均有测试；完全信任也不会绕过基础检查。
- **剩余风险：** Shell、MCP 和进程内插件仍可能利用宿主权限、环境变量或未覆盖的命令形式。
- **处理：** 文档持续明确边界；默认最小权限；未来隔离必须依赖 OS 级机制。
